### 1. 측정 게이트 (Measurement Gate)

#### 펄스 레벨 작동 과정

초전도 트랜스몬 시스템에서 측정은 **분산 읽기(dispersive readout)** 방식으로 이루어진다.
큐빗은 읽기 공명기(readout resonator)에 분산 결합되어 있고,
큐빗 상태(|0⟩ vs |1⟩)에 따라 공명기의 유효 주파수가 달라진다.

```
[측정 자극 펄스]
  MeasureChannel(i)  →  Play(measure_pulse, m_i)
  → 공명기에 마이크로파 자극

[신호 수집]
  AcquireChannel(i)  →  Acquire(duration, a_i, MemorySlot(i))
  → 공명기에서 반사/투과된 신호를 ADC로 디지타이징
```

두 명령은 **반드시 시각 동기화**해야 한다. 멀티플렉스 측정(여러 큐빗 동시)의 경우
각 큐빗마다 Acquire가 동기 정렬되어야 한다 (Sec. 2.3).

#### 3단계 데이터 체인 (Sec. 4)

논문은 측정 데이터를 3단계로 정의하고 사용자가 원하는 레벨을 선택할 수 있게 한다.

```
level-0: raw (원시)
  → ADC가 디지타이징한 복소수 시계열
  → 시간 의존성이 남아있는 벡터

      ↓  [커널 (kernel)]
         이 논문에서는 박스카 적분기(boxcar integrator) 사용
         시계열을 시간 축으로 적분 → 복소수 1개로 압축

level-1: kerneled (커널링 완료)
  → IQ 평면 위의 복소수 점 하나
  → |0⟩과 |1⟩ 대응 점군이 IQ 평면에 클러스터로 분리됨

      ↓  [판별기 (discriminator)]
         이 논문에서는 선형 판별 분석(LDA) 사용
         IQ 점을 결정 경계선 기준으로 분류

level-2: discriminated (판별 완료)
  → 고전 비트 {0, 1}
```

사용자는 level-1 IQ 데이터를 받아 **커스텀 판별기**를 직접 구현할 수 있다.
논문에서는 이를 활용해 ibmq_singapore의 큐빗 16, 17 간 측정 누화(crosstalk)를 조사했다.

#### 논문의 한계 (명시적으로 다루지 않는 것)

- 측정 펄스 자체의 파형 최적화 (duration, amplitude 캘리브레이션)
- 분산 결합의 물리적 세부 이론 (Ref [9] Krantz et al. 로 위임)
- 측정 후 큐빗 상태 붕괴 이후의 재초기화 과정

### 2. 단일 큐빗 게이트 (Single-Qubit Gate)

#### 펄스 레벨 작동 과정

트랜스몬 큐빗은 조셉슨 접합과 커패시터로 구성된 **비선형 LC 발진기**다.
|0⟩ ↔ |1⟩ 전이를 |1⟩ ↔ |2⟩ 전이와 구별할 수 있는 것은 **비조화성(anharmonicity) δ** 덕분이다.

- ibmq_almaden 제어 큐빗(큐빗 1): f₁ = 4.972 GHz, δ₁ = −319.7 MHz
- ibmq_almaden 타겟 큐빗(큐빗 0): f₀ = 4.857 GHz, δ₀ = −320.2 MHz

```
DriveChannel(i)  →  Play(pulse, d_i)
  → 큐빗 i의 공명 주파수 f_i 로 변조된 마이크로파 출력
  → 블로흐 구면(Bloch sphere)에서 큐빗 상태를 회전
```

실제 출력 신호: Dⱼ = Re[e^{i2πfjdt + φ} dⱼ]
- dⱼ: 펄스 샘플 (복소 엔벨로프)
- f: DriveChannel의 주파수 (큐빗 공명 주파수로 설정)
- φ: 채널 위상 (ShiftPhase로 누적)

#### 사용되는 펄스 종류

| 게이트 | 펄스 종류 | 설명 |
|--------|-----------|------|
| X, X/2 회전 | **DRAG** (Derivative Removal via Adiabatic Gate) | Gaussian + 미분 성분(직교 위상) 혼합. 누설 억제. |
| Z 회전 | **ShiftPhase** (Virtual Z-gate) | 하드웨어 펄스 없음. 채널 위상 φ를 소프트웨어로 누적. |

**Virtual Z-gate (Sec. 2.2, Ref [19,20]):**
```python
ShiftPhase(phase=θ, channel=DriveChannel(i))
```
- 지속 시간: 0 (instantaneous)
- 하드웨어 AWG 출력 없음
- 이후 모든 펄스의 위상이 θ만큼 회전하는 효과
- 원리적으로 에러가 없는 이상적 게이트

### 3. 2큐빗 게이트 — Cross-Resonance (CR) 게이트

#### 펄스 레벨 작동 과정 (Sec. 3.1)

CR 게이트의 핵심: **제어 큐빗**을 **타겟 큐빗의 공명 주파수**로 구동.

```
ControlChannel(u_i)  →  Play(cr_pulse, u_i)
  → 제어 큐빗에 타겟 주파수(f_target)의 마이크로파 인가
  → 두 큐빗 간 분산 결합을 통해 유효 ZX 상호작용 유도
```

유효 해밀토니안 (2차 섭동론, Ref [28]):

```
H_CR = (Z⊗B)/2 + (I⊗C)/2

B = ω_ZI·I + ω_ZX·X + ω_ZY·Y + ω_ZZ·Z
C = ω_IX·X + ω_IY·Y + ω_IZ·Z
```

이상적으로는 ZX 항만 남겨 아래 게이트를 구현:

```
U_ZX(θ_ZX) = exp(-i·θ_ZX·ZX/2)
```

θ_ZX = π/2일 때 → **perfect entangler** → 국소 단일 큐빗 회전과 함께 CNOT 구현 가능.

#### CR1 vs CR2 시퀀스

**CR1 (단일 펄스):**
```
ControlChannel(u₁):  [  CR_pulse (+amp)  ]
```

**CR2 (에코 2펄스):**
```
ControlChannel(u₁):  [ CR_pulse (+amp) ]  [ CR_pulse (-amp) ]
DriveChannel(d₁):               [ X₁₈₀ ]               [ X₁₈₀ ]
```
에코 펄스(X₁₈₀)가 ZI, IX, IY 항의 부호를 반전시켜 상쇄함.

### 4. 단일 큐빗 게이트 에러 (논문 기준)

논문에서 단일 큐빗 에러는 2큐빗 게이트만큼 체계적으로 분석되지 않는다.
언급된 내용만 정리한다.

#### 4.1 누설 에러 (Leakage Error)

- **원인**: |0⟩ ↔ |1⟩ 전이 펄스가 |1⟩ ↔ |2⟩ 전이도 자극
- **조건**: 비조화성 δ ≈ −320 MHz. 펄스가 너무 짧으면(대역폭 ↑) 인접 전이 자극 가능
- **대응**: **DRAG 펄스** (Derivative Removal via Adiabatic Gate)
  - Gaussian 엔벨로프의 미분을 직교 위상(Q)에 혼합
  - |2⟩ 상태로의 전이 진폭을 상쇄
  - 논문에서 단일 큐빗 π 펄스(x180)와 π/2 펄스(x90) 모두 DRAG 형태 사용

#### 4.2 T₁, T₂ 감쇠 에러 (Relaxation Error)

- **원인**: 큐빗이 대기 시간 동안 환경과 상호작용하여 에너지 이완(T₁) 및 위상 디코히어런스(T₂) 발생
- **대응**: 스케줄러의 **as-late-as-possible (ALAP)** 정책 (Sec. 2.5)
  - 측정 직전까지 큐빗을 가능한 한 기저 상태 |0⟩에 유지
  - 첫 펄스와 측정 사이의 유효 대기 시간 최소화

#### 4.3 위상 에러 → Virtual Z-gate로 완전 제거

- ShiftPhase는 하드웨어 펄스를 출력하지 않으므로 원리적으로 에러 없음
- Z 회전이 필요한 모든 상황에서 물리 펄스 대신 가상 Z 사용

#### 논문에서 다루지 않는 단일 큐빗 에러

- 피델리티 수치: 백엔드 제공값 사용 (논문에서 독립 측정 안 함)
- 제어 펄스 형태 최적화 (amplitude, duration 튜닝)
- 랜덤 벤치마킹(RB)에 의한 단일 큐빗 게이트 에러율 (2큐빗 RB에 포함되어 간접적으로만 반영)

### 5. 2큐빗 게이트 에러 (논문 기준, 상세)

2큐빗 에러는 논문의 핵심 주제로, 체계적으로 다룬다.

#### 5.1 에러 원인: 유효 해밀토니안의 불필요한 항

목표는 ZX 항만 남기는 것이지만, 실제로는 여러 항이 동시에 존재한다.

| 항 | 물리적 의미 | CR1 크기 | CR2(에코) 크기 | 억제 방법 |
|----|------------|:-------:|:-------------|---------|
| **ZX** | 목표 상호작용 | 존재 | 존재 | — (목표) |
| **ZI** | 제어 큐빗만 회전 (단일 큐빗 오류) | **매우 큼** (타 항의 ~100배, 1/100 스케일로 표시) | 에코로 거의 소거 | 에코 시퀀스 |
| **IX, IY** | 타겟 큐빗만 회전 (단일 큐빗 오류) | 존재 | 에코로 소거 | 에코 시퀀스 |
| **ZY** | ZX와 유사하나 축 틀어짐 | 존재 | 존재 (에코 후 잔류) | 위상 보정 φ_opt |
| **ZZ** | 조건부 위상 오류 | 잔류 | 잔류 | 국소 회전으로 부분 보정 |

#### 5.2 측정·분석 방법: QPT + Lindblad 초연산자 (Sec. 3.3, Appendix C)

에러 항들의 크기를 **정량적으로 측정**하는 흐름:

```
1. 각 CR 펄스 진폭 A에서 QPT 수행
   → 144개 회로 (4개 입력 상태 × 3개 측정 기저 × 두 큐빗²), 각 2048 샷

2. Choi 행렬 Λ_ε 재구성
   → 최대 우도 추정 (convex optimization), CPTP 보장

3. 초연산자 변환: Λ_ε → S_ε

4. 행렬 로그로 Lindblad 생성자 추출
   S_G = t⁻¹ log(S_ε)

5. 파울리 기저 직교성 활용하여 계수 분리
   ω_ij = Tr[S_{L_{Bij}}† S_G] / ‖S_{L_{Bij}}‖

   핵심: S_L (해밀토니안 부분)과 S_D (소산 부분)은 직교
   → 노이즈 있어도 해밀토니안 항만 깨끗하게 추출 가능
```

**결과 (Figure 5):**
- CR1: ZX 단조 증가, ZI가 압도적으로 큼, IX/IY 존재
- CR2: ZX 단조 증가, ZI ≈ 0, IX/IY ≈ 0 (에코 효과 확인)
- Hamiltonian 재구성 프로세스 피델리티: CR1 99.4%, CR2 98.6% (평균)

#### 5.3 에러 억제 단계별 흐름

**Step 1. 위상 보정 (Appendix B) — ZY 항 제거**

실온 전자장비와 큐빗 사이 동축 케이블의 전달 함수가 위상 오프셋 φ₀ 유발 → ZY 항 발생.

CR2 에코 해밀토니안 근사:
```
H_CR ≈ Ω(A,φ)(cos φ₀ · ZX + sin φ₀ · ZY) + ε
```

보정 절차:
1. |00⟩ 초기화 후 CR2 진폭 스윕 → Y 기저에서 π/2 회전 달성 진폭 A_opt = 0.108 탐색
2. 제어 큐빗 상태 |0⟩, |1⟩ 각각에서 Y 기저 신호를 φ로 스윕
3. |Tr(σ_y ρ)| 최대화하는 위상: **φ_opt = −0.166 rad**

**Step 2. 진폭 보정 (Appendix C) — θ_ZX = π/2 달성**

3차 섭동 전개로 ZX 계수의 진폭 의존성 피팅:

```
ω_ZX(Ā)/2 = -Jλ Ā/Δ · (δ₁/(δ₁+Δ)) + [3차 항]
```

피팅 파라미터:
- 결합 강도 J = 1.87 ± 0.046 MHz (CR1), 1.79 ± 0.033 MHz (CR2)
- θ_ZX = π/2 달성 진폭: CR1 = 0.229 ± 0.019, CR2 = 0.098 ± 0.005

**Step 3. 국소 단일 큐빗 회전 최적화 (Appendix D) — 잔류 결맞음 에러 제거**

CR 게이트 전후에 단일 큐빗 회전을 추가해 CNOT으로 변환:

```
F_max = max_Θ F[ε_CR(Ā_{π/2}), U_ent(Θ)]
```

12개 실수 파라미터 Θ (제어·타겟 각각 전·후 U₃ 회전):
```
U_ent(Θ) = U_pre†(Θ) · U_ent · U_post†(Θ)
U_pre(Θ) = U₃(Θ₀,Θ₁,Θ₂) ⊗ U₃(Θ₃,Θ₄,Θ₅)
U_post(Θ) = U₃(Θ₆,Θ₇,Θ₈) ⊗ U₃(Θ₉,Θ₁₀,Θ₁₁)
```

최적화 후 이론 피델리티: CR1 F_max = 0.992, CR2 F_max = 0.994

**Step 4. 측정 에러 보정 — 판독 에러 완화**

QPT 진행 중 측정 에러도 보정:
- 4개 보정 스케줄(|00⟩, |01⟩, |10⟩, |11⟩ 준비) 추가 실행
- Qiskit Ignis readout error mitigation 적용
- QPT 회로 사이에 인터리빙

#### 5.4 최종 피델리티 결과

랜덤 벤치마킹(RB)으로 측정한 실제 평균 게이트 피델리티:

| 게이트 | 캘리브레이션 방법 | 측정 피델리티 |
|--------|-----------------|:------------:|
| CR1-CNOT | 오픈 루프, 취소 톤 없음 | **F = 0.981** |
| CR2-CNOT | 오픈 루프, 취소 톤 없음 | **F = 0.979** |
| 백엔드 기본 CNOT | 폐루프, 에코 + 취소 톤 포함 | F = 0.984 |

> 취소 톤(cancellation tone)과 폐루프 캘리브레이션 없이도 백엔드 기본값과 거의 동일한 피델리티 달성.
> 이론 최대 대비 ~1% 손실은 단일 큐빗 게이트 자체의 불완전성에서 기인 (논문 명시).

### 2-1. 트랜스몬-공명기 결합: Jaynes-Cummings 해밀토니안

트랜스몬 큐빗과 읽기 공명기(readout resonator)는 **용량성 결합(capacitive coupling)** 으로 연결된다.
두 계를 합친 전체 해밀토니안은 **Jaynes-Cummings 모델**로 기술된다:

```
H_JC = ℏω_r (a†a + 1/2)          → 공명기 (조화 발진기)
      + ℏω_q/2 · σ_z              → 큐빗 (2준위계)
      + ℏg (a†σ⁻ + a σ⁺)         → 결합 항 (광자 교환)
```

- `ω_r`: 공명기 고유 주파수
- `ω_q`: 큐빗 전이 주파수 (|0⟩ ↔ |1⟩)
- `g`: 결합 강도 (보통 수십~수백 MHz 수준)
- `a†, a`: 공명기 광자 생성·소멸 연산자
- `σ⁺, σ⁻`: 큐빗 올리기·내리기 연산자

이 해밀토니안은 큐빗과 공명기가 광자를 실제로 교환하는 **공명(resonant) 결합** 을 기술한다.

---

### 2-2. 분산 결합 (Dispersive Regime)

측정 중에 큐빗이 광자를 흡수해 상태가 바뀌면 측정이 파괴된다(QND 조건 위반).
이를 피하기 위해 공명기 주파수와 큐빗 주파수를 **크게 이탈(detuning)** 시킨다:

```
Δ = ω_q - ω_r,   |Δ| >> g
```

이 **분산 결합 조건(dispersive regime)** 에서 `g/|Δ| << 1`이 되어
큐빗과 공명기 사이의 실제 광자 교환이 억제된다.

이 조건에서 유니터리 변환(Schrieffer-Wolff 변환)을 적용하면
Jaynes-Cummings 해밀토니안이 아래의 **분산 해밀토니안**으로 근사된다:

```
H_disp = ℏω_r a†a  +  ℏ(ω_q/2) σ_z  +  ℏχ a†a σ_z
```

새로 등장한 항: `ℏχ a†a σ_z`

여기서 **분산 이동(dispersive shift)** χ는:
```
χ ≈ g² / Δ       (단순 2준위 근사)
```

트랜스몬의 경우 |2⟩ 준위 효과까지 포함하면:
```
χ = g² δ / [Δ(Δ + δ)]
```
- `δ`: 트랜스몬 비조화성 (anharmonicity, ≈ −320 MHz)
- `Δ`: 큐빗-공명기 이탈량

---

### 2-3. 트랜스몬 상태에 따른 공명기 유효 주파수 변화

`H_disp`의 `χ a†a σ_z` 항을 큐빗 상태 고유값으로 분리하면:

**큐빗가 |0⟩ (σ_z 고유값 = +1)일 때:**
```
H_disp|₀ = ℏ(ω_r + χ) a†a  +  상수
```
→ 공명기가 주파수 **ω_r + χ** 로 동작

**큐빗가 |1⟩ (σ_z 고유값 = −1)일 때:**
```
H_disp|₁ = ℏ(ω_r − χ) a†a  +  상수
```
→ 공명기가 주파수 **ω_r − χ** 로 동작

```
큐빗 상태:    |0⟩                    |1⟩

공명기 주파수:  ω_r + χ               ω_r − χ
                  ↑                      ↑
               (높은 쪽)             (낮은 쪽)
                     |←── 2χ ──→|
```

즉, 큐빗 상태에 따라 공명기 주파수가 **±χ 씩 이동**한다.
두 상태의 공명기 주파수 차이는 **2χ** 이며, 이것이 측정의 구별 가능성을 결정한다.

---

### 2-4. 실제 측정 과정: 탐침 펄스와 IQ 신호

공명기 주파수가 큐빗 상태에 따라 달라진다는 것을 이용해 큐빗 상태를 읽어낸다.

#### 탐침 펄스 인가

```
MeasureChannel(i)  →  공명기를 ω_probe 주파수로 구동
```

탐침 주파수 ω_probe는 ω_r + χ와 ω_r − χ 사이 어딘가로 설정한다.
(보통 ω_r 근방 또는 두 주파수의 중간)

#### 공명기의 주파수 의존 응답

공명기는 입력 신호를 **반사 또는 투과**시킨다.
입력 신호와 출력 신호 사이의 **위상 차이(phase shift)** 와 **진폭 변화**는
공명기의 유효 주파수에 따라 달라진다.

```
큐빗 = |0⟩  →  공명기 주파수 ω_r + χ
             →  탐침 신호가 이 주파수의 로렌츠 곡선 한쪽에서 반사
             →  반사 신호에 위상 θ₀ 부여

큐빗 = |1⟩  →  공명기 주파수 ω_r − χ
             →  탐침 신호가 반대 쪽에서 반사
             →  반사 신호에 위상 θ₁ 부여   (θ₁ ≠ θ₀)
```

이상적으로 2χ >> κ (κ: 공명기 선폭) 이면 두 반사 신호가 완전히 구별된다.

#### IQ 복조 (Demodulation)

반사 신호는 ADC에서 디지타이징된 후 반송파(carrier)와 믹싱하여 복조된다.
복조 결과는 **I (동위상)** 와 **Q (직교위상)** 성분으로 분리된다.

```
복소 신호 s(t) = I(t) + iQ(t)
```

박스카 적분기(boxcar integrator)로 시간 적분하면 IQ 평면의 점 하나가 된다:

```
큐빗 = |0⟩  →  IQ 평면의 점군 A (클러스터)
큐빗 = |1⟩  →  IQ 평면의 점군 B (클러스터)
```

```
Q축
 ↑
 |       ● ● ●  ← |0⟩ 클러스터 (A)
 |     ●●●●●
 |
 |                 ○ ○ ○  ← |1⟩ 클러스터 (B)
 |               ○ ○ ○ ○
 +─────────────────────────→ I축
```

판별기(discriminator)는 이 두 클러스터 사이에 결정 경계(decision boundary)를 긋는다.
논문에서는 **선형 판별 분석(LDA)** 을 사용한다.

---

### 2-5. QND(Quantum Non-Demolition) 측정 조건

분산 결합에서 측정이 QND인 이유:
- 분산 해밀토니안 `H_disp`에서 `[H_disp, σ_z] = 0`
- 즉, 측정 과정이 큐빗의 σ_z 고유값을 보존함
- 공명기 광자가 큐빗 상태를 바꾸지 않고 큐빗 상태 **정보만 추출**

단, 실제 트랜스몬에서는:
- 탐침 광자 수가 많아지면(고출력 측정) |2⟩ 이상 준위 전이 유발 → QND 파괴
- 측정과 함께 발생하는 Purcell 감쇠: 공명기를 통한 큐빗 에너지 이완 경로

## [논의 3] 중첩 상태에서의 측정: 공명기 얽힘과 파동함수 붕괴

> 질문 A: 중첩 상태 (|0⟩+|1⟩)/√2 에서 측정하면 50% 확률로 +χ, 50% 확률로 -χ 이동이 생기는 건가?
> 질문 B: 측정으로 결과가 나오면 큐빗이 실제로 그 상태로 고정되는가?
> (+χ shift → 큐빗이 |0⟩으로 붕괴하는 식으로?)

---

### 3-1. 중첩 상태에서의 공명기 거동: 얽힘이 먼저다

**직관적 이해 (통계적으로는 맞지만 물리적으로 불완전):**
"50% 확률로 +χ, 50% 확률로 -χ" — 많은 샷을 반복하면 결과적으로 맞다.

**양자역학적으로 정확한 과정:**

공명기에 탐침 펄스가 들어오면, 분산 해밀토니안이 큐빗과 공명기를 **얽히게** 만든다.
"공명기가 먼저 어느 한쪽으로 확 당겨지는 것"이 아니다.

```
초기 상태:
|ψ_초기⟩ = (|0⟩ + |1⟩)/√2  ⊗  |α_in⟩

시간 진화 후 (탐침 펄스 동안):
|ψ_얽힘⟩ = 1/√2 ( |0⟩⊗|α₀⟩  +  |1⟩⊗|α₁⟩ )
```

- `|α₀⟩`: 공명기가 ω_r+χ 로 진동하는 결맞음 상태 → IQ 평면에서 A 방향으로 회전
- `|α₁⟩`: 공명기가 ω_r-χ 로 진동하는 결맞음 상태 → IQ 평면에서 B 방향으로 회전

이 시점에서 공명기는 두 상태의 **양자 중첩(얽힘 상태)** 에 있다.
"+χ도 아니고 -χ도 아닌" 상태다. 고전적 유추로는 설명할 수 없다.

**ADC가 IQ 신호를 읽는 순간 → 붕괴:**

```
IQ 점이 A 클러스터에 찍힘  →  |ψ_후⟩ = |0⟩  (확률 1/2)
IQ 점이 B 클러스터에 찍힘  →  |ψ_후⟩ = |1⟩  (확률 1/2)
```

즉, "50%로 +χ, 50%로 -χ"라는 표현은 **결과를 통계적으로 해석할 때** 맞는 말이고,
과정으로는 "공명기가 큐빗과 얽혀 있다가, IQ를 읽는 순간 둘 다 동시에 붕괴한다"가 정확하다.

---

### 3-2. 측정 후 큐빗 상태: 붕괴와 고정

**결론부터: 예, +χ shift → |0⟩으로 붕괴하여 고정된다. (이상적 경우)**

이것은 양자역학의 **투영 가정(projection postulate)** 이다.
측정 결과 m이 나오면 큐빗은 해당 고유 상태 |m⟩으로 붕괴하고,
이후 큐빗은 |m⟩ 상태를 유지한다.

**분산 측정에서 이것이 보장되는 이유 — QND 조건:**

분산 해밀토니안에서:
```
[H_disp, σ_z] = 0
```

측정 해밀토니안(`χ a†a σ_z`)이 큐빗의 관측량(`σ_z`)과 교환 가능하다.
→ 측정 과정 자체가 큐빗의 σ_z 고유값을 바꾸지 않는다.
→ "측정이 큐빗 상태를 교란하지 않는다" = **QND(Quantum Non-Demolition) 측정**

**QND 측정의 실험적 의미:**

```
1회차 측정: (|0⟩+|1⟩)/√2  →  결과 |0⟩
2회차 측정 (즉시):           →  결과 |0⟩  (반드시)
3회차 측정:                  →  결과 |0⟩  (반드시)
```

첫 번째 측정이 큐빗을 |0⟩으로 붕괴시키고, 이후 반복 측정은 계속 |0⟩을 반환한다.
이것이 성립하면 "측정이 QND하다"고 말한다.

---

### 3-3. 이상 vs 현실: 붕괴가 항상 완벽하지 않은 이유

실제 실험에서는 여러 불완전성이 존재한다.
에러를 두 종류로 구분하는 것이 중요하다.

#### 종류 A: 판독 분류 에러 (Readout Classification Error)

큐빗은 실제로 정확히 붕괴했지만, IQ 신호를 잘못 분류하는 에러.

```
큐빗이 진짜 |0⟩  →  공명기 신호가 A 클러스터 근처에 찍힘
                   →  그런데 노이즈로 인해 IQ 점이 결정 경계 너머로 넘어감
                   →  판별기가 |1⟩으로 오분류  →  잘못된 측정 결과
```

이것은 큐빗 상태 붕괴 자체는 올바르게 일어났지만, **우리가 읽은 값이 틀린** 경우다.
원인: 열잡음, 앰프 노이즈, ADC 양자화 잡음.

이 논문의 측정 배정 피델리티 `F_a = 1 - (Pr[0|1] + Pr[1|0])/2`:

| 큐빗 | 판별기 종류 | F_a |
|------|-----------|:---:|
| Q16 | Single | 89.21% |
| Q17 | Single | 91.06% |

→ Q16 기준으로 약 10.8%의 확률로 잘못 읽힌다.
이것이 바로 판독 분류 에러다.

#### 종류 B: 측정 역작용 에러 (Measurement Backaction Error)

측정 과정 자체가 큐빗 상태를 교란하는 에러. QND 조건이 깨지는 경우.

**원인들:**

| 원인 | 내용 |
|------|------|
| 고출력 측정 (dressing effect) | 탐침 광자 수가 많아지면 큐빗-공명기 결합이 비선형 영역 진입 → `[H, σ_z] ≠ 0` |
| Purcell 감쇠 | 공명기를 통한 큐빗 에너지 이완 경로. 측정 중 큐빗이 |1⟩→|0⟩으로 자발 붕괴 |
| 측정 유도 상태 전이 (MIST) | 탐침 광자가 트랜스몬의 높은 준위를 자극 → 측정 도중 큐빗 상태 변화 |

이 에러가 있으면 "측정 결과 |0⟩이 나왔지만 실제 큐빗은 |0⟩이 아닐 수 있다"는 상황이 발생한다.

**두 에러의 비교:**

```
판독 분류 에러:   큐빗 상태는 올바르게 붕괴  →  우리가 잘못 읽음
측정 역작용 에러: 우리는 올바르게 읽음        →  큐빗 상태가 잘못 붕괴
```

---

### 3-4. 이 논문과의 연결

Alexander et al.은 이 두 에러를 직접 구분해서 분석하지는 않는다.
하지만 관련 내용이 간접적으로 등장한다:

| 내용 | 논문 위치 |
|------|-----------|
| 측정 배정 피델리티 F_a 정의 및 측정 | Sec. 4, Table 2 |
| Qiskit Ignis readout error mitigation 적용 | Sec. 3.3 |
| 보정 스케줄 4개 (|00⟩,|01⟩,|10⟩,|11⟩) 추가 실행 | Sec. 3.3 |
| 측정 누화 없음 확인 (Pearson t-검정) | Sec. 4 |

논문이 적용한 **readout error mitigation**은 판독 분류 에러(종류 A)를 통계적으로 보정한다.
측정 배정 행렬 M을 구성해:
```
p_measured = M · p_true  →  p_true = M⁻¹ · p_measured
```
이 보정으로 QPT 결과의 정확도를 높인다.

### 3-5. 전체 흐름 정리

```
큐빗: (|0⟩+|1⟩)/√2
         │
         │  분산 해밀토니안 H_disp = ℏχ a†a σ_z
         ▼
큐빗+공명기 얽힘: 1/√2 (|0⟩⊗|α₀⟩ + |1⟩⊗|α₁⟩)
   [이 시점에서 공명기는 +χ도 -χ도 아닌 중첩]
         │
         │  ADC로 IQ 신호 읽기 (= 공명기 측정)
         ▼
   A 클러스터 (50%)       B 클러스터 (50%)
        │                      │
        ▼                      ▼
  큐빗: |0⟩             큐빗: |1⟩
  (투영 가정에 의해 붕괴)  (투영 가정에 의해 붕괴)
        │                      │
        │  이후 즉시 재측정하면?
        ▼
  |0⟩ 반복 확인 가능    |1⟩ 반복 확인 가능
  (QND 조건 성립 시)
```

**핵심 요약:**
- 측정 전: 공명기와 큐빗이 얽힘. 공명기는 +χ/-χ 중첩 상태.
- IQ를 읽는 순간: 둘 다 붕괴. 큐빗은 |0⟩ 또는 |1⟩로 고정.
- 이상적 QND에서: 붕괴 후 큐빗 상태는 측정 결과와 100% 일치하며 유지됨.
- 현실에서: 판독 분류 에러(~10%)와 측정 역작용 에러가 이 이상을 깨뜨림.

### 4-1. 마이크로파 펄스의 I·Q 성분

#### I, Q란 무엇인가

마이크로파 신호는 수 GHz의 반송파(carrier)를 갖는다. 이 반송파를 직접 다루면 계산이 복잡하므로, 반송파 주파수 기준으로 천천히 변하는 **복소 엔벨로프(complex envelope)** 만 다룬다.

```
실제 출력 전압:
  V(t) = I(t) · cos(ω_c · t) − Q(t) · sin(ω_c · t)
       = Re[ (I(t) + i·Q(t)) · e^{i·ω_c·t} ]
       = Re[ A(t) · e^{i·ω_c·t} ]

A(t) = I(t) + i·Q(t)  ← 복소 엔벨로프
```

- **I(t)**: In-phase. 반송파와 같은 위상의 성분. (코사인 축)
- **Q(t)**: Quadrature. 반송파보다 90° 앞선 성분. (사인 축)
- 반송파 ω_c는 MeasureChannel 주파수 (≈ 7 GHz)
- I, Q는 마이크로파 **전압 펄스의 엔벨로프** 이다 (수 GHz 오실레이션 자체가 아님)

#### 순수 Gaussian 측정 펄스에서 Q = 0인 이유

시뮬레이터의 측정 펄스는 실수 Gaussian이다:

```python
pulse = gaussian_pulse(...)   # 복소수 배열, 실제로는 실수부만 사용
I(t) = A · exp(−(t−t₀)² / (2σ²))
Q(t) = 0
```

Q = 0 이라는 것은 펄스의 위상이 반송파와 완전히 정렬되어 있다는 뜻이다.
DRAG 펄스(Phase 2)에서는 Q(t) = β · dI/dt ≠ 0 이 추가된다.

#### Qiskit Pulse에서의 표현

Qiskit Pulse에서 각 시간 샘플 `dⱼ`는 복소수다:

```
dⱼ = Iⱼ + i·Qⱼ

실제 AWG 출력:
  D(t) = Re[ dⱼ · e^{i(2π·f·j·DT + φ)} ]
```

- `f`: DriveChannel/MeasureChannel의 주파수 (LO 주파수)
- `φ`: ShiftPhase로 누적된 위상
- `DT = 0.222 ns`: 한 샘플의 시간 간격 (ibmq_almaden 기준)

### 4-2. S11(ω) 반사 스펙트럼

#### S11이란

S11은 마이크로파 회로의 **반사 계수(reflection coefficient)** 다.
전송 라인에서 신호를 쏘았을 때 얼마나 반사되어 돌아오는지를 주파수의 함수로 나타낸다.

```
S11(ω) = V_reflected(ω) / V_incident(ω)
```

이상적 공명기(internal loss 없음):
- 공명 주파수 ω_r에서 에너지가 모두 공명기에 흡수됐다가 반사됨
- 모든 주파수에서 |S11| = 1 (진폭은 변하지 않음)
- 대신 **위상** φ(ω)가 ω_r 근방에서 0° → ±180° 로 빠르게 변함

실제 공명기(internal loss κ_int 있음):
- ω_r에서 |S11| < 1 (일부 에너지가 내부 손실로 사라짐)

#### 시뮬레이터에서 보여주는 것

시뮬레이터 패널 [2]는 |S11|의 위상 변화 대신 **파워 결합 Lorentzian** 을 표시한다:

```
L(ω) = (κ/2)² / ((ω − ω_r)² + (κ/2)²)

- ω_r에서 최대값 1 (공명기와 최대 결합)
- FWHM = κ  (공명기 선폭)
- 멀리 갈수록 → 0
```

이 Lorentzian은 물리적으로 "탐침 주파수 ω에서 공명기가 얼마나 응답하는가"를 나타낸다.

#### 큐빗 상태에 따른 분리

분산 결합에 의해 공명기 주파수가 큐빗 상태별로 이동하므로, 두 Lorentzian이 나타난다:

```
|0⟩: L₀(ω) peaked at ω_r + χ
|1⟩: L₁(ω) peaked at ω_r − χ

탐침 주파수: ω_probe = ω_r (두 peak의 중간)
```

탐침 주파수에서 두 곡선의 L 값 차이가 클수록, 반사 신호의 진폭/위상 차이가 커져 IQ 클러스터 간 거리가 늘어난다.
이 차이는 χ = κ/2 일 때 최대가 된다 (시뮬레이터 패널 [2]의 chi_opt 표시).

### 4-3. 패널 [3] 시간 도메인 IQ — 버그 수정 기록

#### 현상

- 패널 [1] (측정 펄스): 시간 축 0 → 0.44 us (MEAS_DURATION = 2000 사이클)
- 패널 [3] (시간 도메인 IQ): 시간 축 0 → 0.11 us (500 사이클 하드코딩)

#### 원인

`viz_panel.py`에서 패널 [3]의 시뮬레이션 길이가 `duration=500`으로 하드코딩되어 있었다.
측정 펄스 duration(MEAS_DURATION = 2000)과 무관한 값이었다.

#### 수정

```python
# 수정 전
_T_TD_US = np.arange(500) * config.DT * 1e6
generate_time_domain_iq(..., duration=500, ...)

# 수정 후
_T_TD_US = _T_PULSE_US   # MEAS_DURATION과 동일
generate_time_domain_iq(..., duration=config.MEAS_DURATION, ...)
```

패널 [3]이 이제 패널 [1]과 같은 시간 범위(0 → 0.44 us)를 보여준다.

#### 물리적 의미

패널 [3]에서 관찰되는 **ring-up** 과도 응답:

```
t = 0: I(t) = Q(t) = 0  (공명기 내부 에너지 없음)
         ↓
공명기에 탐침 에너지가 서서히 채워짐 (ring-up)
         ↓
t >> 1/κ: 정상 상태 (steady state) 도달
  → I(t), Q(t)가 일정한 값으로 수렴
  → 이 값이 IQ 산점도의 한 점이 됨
```

ring-up 시간상수: τ = 1/κ = 1/(0.5 MHz) = 2 μs (기본값 기준)
기본 측정 duration이 0.44 μs이므로 정상 상태에 도달하기 전에 측정이 끝나는 것을 확인할 수 있다.
κ를 크게 늘리면 ring-up이 빨리 완료되는 것을 패널 [3]에서 직접 관찰할 수 있다.

### 4-4. 측정 노이즈: Gaussian이 주된 것인가?

#### 시뮬레이터의 노이즈 모델

시뮬레이터는 IQ 신호에 **독립적인 Gaussian 노이즈** 를 더한다:

```python
I_measured = I_signal + N(0, noise_sigma²)
Q_measured = Q_signal + N(0, noise_sigma²)
```

이것은 **증폭기 열잡음(amplifier thermal noise)** 을 모델링한다.

#### 현실의 주요 노이즈 원천

| 노이즈 원천 | 분포 | 설명 |
|------------|------|------|
| **HEMT 증폭기 열잡음** | Gaussian (I, Q 독립) | 4 K에서 동작. ~20 noise photon 추가. **현재 실험의 지배적 노이즈.** |
| **광자 산탄 잡음(photon shot noise)** | Gaussian | 측정 토닌 자체의 양자 요동. coherent state → Gaussian IQ 분포. |
| **파라메트릭 증폭기(JPA/TWPA) 이후** | Gaussian (양자 한계에 가까움) | HEMT 앞에 배치하면 노이즈를 ~0.5 photon 수준으로 억제. |
| **T₁ 감쇠 (에너지 이완)** | 비Gaussian (지수 감쇠) | 측정 중 큐빗이 |1⟩→|0⟩으로 자발 붕괴. IQ 점이 두 클러스터 사이에 찍힘. |
| **MIST (측정 유도 상태 전이)** | 비Gaussian | 탐침 광자가 |2⟩ 이상 준위를 자극 → 상태 전이. 고출력 측정에서 발생. |

#### 결론: Gaussian은 주된 노이즈이지만 전부는 아니다

**Gaussian이 맞는 경우:**
- HEMT + 파라메트릭 증폭기 체인이 지배하는 영역
- 측정 시간이 T₁보다 훨씬 짧을 때 (T₁ 효과 무시 가능)
- 탐침 광자 수가 적당할 때 (MIST 없음)

이 조건에서는 IQ 산점도의 각 클러스터가 2D Gaussian 분포를 이루며, LDA(선형 판별 분석)가 최적 분류기가 된다.

**Gaussian이 부족한 경우:**

```
T₁ 감쇠:
  큐빗 |1⟩이 측정 중간에 |0⟩으로 붕괴
  → I(t), Q(t)가 ring-up 도중에 클러스터 A → B로 점프
  → 적분된 IQ 값이 두 클러스터 사이 어딘가에 찍힘
  → 꼬리(tail)가 긴 비Gaussian 분포
```

이 경우 LDA보다 **신경망(NN) 분류기** 나 **히스토그램 기반 임계값** 이 유리하다.
Alexander et al. (2020)에서 LDA를 사용하는 것은 단순성과 실용성 때문이며,
T₁이 충분히 길고 노이즈가 Gaussian에 가깝다는 가정 하에 적용된다.

#### 시뮬레이터 한계

현재 시뮬레이터는 Gaussian 노이즈만 구현했고, T₁ 감쇠와 MIST는 포함하지 않는다.
실제 실험 데이터에서는 IQ 클러스터가 완벽한 Gaussian이 아닌 경우가 많고,
이것이 측정 피델리티 < 99%의 한 원인이다.
