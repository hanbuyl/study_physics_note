# 추가 논의 노트: 실험적 관점에서의 Pauli Twirling

> 논문 `s41598-019-46722-7` 학습 이후 심화 논의 정리

---

## 1. 실제 노이즈의 파울리 기저 V는 어떻게 알아내는가

논문은 V(노이즈의 파울리 기저)를 안다고 전제하고 시작하지만, 실험적으로는 이것 자체가 핵심 문제다.

### 닭이냐 달걀이냐 문제

```
Reduced twirling을 쓰려면 V를 알아야 함
        ↓
V를 알려면 노이즈를 먼저 측정해야 함
        ↓
노이즈를 측정하려면... 트월링이 필요할 수도 있음
```

### 해결 방법 1: 물리적 모델링 (가장 현실적)

하드웨어 특성으로 V를 예측한다. 논문의 Steane 코드 예시도 이 방법이다.

| 하드웨어 | 지배적 노이즈 | 예측 V |
|---|---|---|
| 초전도 큐비트 | 위상 이완 (T₂) | {I, Z} 위주 |
| 스핀 큐비트 | Z 위주 탈위상 | {I, Z} 거의 순수 |
| 이온 트랩 | 글로벌 위상 노이즈 | {I, Z₁, ..., Zₙ} |
| CZ 게이트 잔류 결합 | ZZ 커플링 | {I, Z₀Z₁} |

### 해결 방법 2: Quantum Process Tomography (QPT)

게이트를 완전히 특성화해서 V를 추출하지만, 큐비트 수가 늘면 비현실적이다.

```
필요한 실험 횟수: 4^(2n)
  n=2: 256회
  n=7: 약 268억회 ← 완전히 비현실적
```

### 해결 방법 3: Sparse Pauli Learning (현재 연구 최전선)

V가 sparse할 것이라는 가정 하에 효율적으로 추정한다. (Harper et al. 2020, Flammia & O'Brien 2020)

```
필요한 실험 횟수:
  QPT:                O(4^(2n))
  Sparse Pauli 학습:  O(|V| · n²)  ← V가 작으면 훨씬 적음
```

### 실용적 워크플로우

```
[1단계] 물리적 모델로 V 예측
        "이 하드웨어는 Z 노이즈 위주일 것" → V_guess

[2단계] Ramsey/Rabi 실험으로 빠른 검증
        Z 에러 지배적 확인 → V_guess 채택

[3단계] V 기반으로 Reduced W 설계 (논문 방법 적용)

[4단계] Reduced W로 트월링 후 결과 측정

[5단계] Full twirling 결과와 비교 (validation)
        다르면 V에 누락된 에러 존재 → V 확장 후 반복
```

## 2. 노이즈 정상성 가정 (Noise Stationarity)

V를 한 번 측정한 뒤 미래에도 같은 구조일 것이라 가정하고 Reduced W를 사용한다.

```
[특성화 단계]  오늘 노이즈를 측정해서 V를 파악
      ↓         "이 하드웨어는 Z 노이즈 위주다"
[실험 단계]    내일도 그럴 것이라 가정하고 Reduced W 사용
```

### 이 가정이 깨지는 경우

| 원인 | 현상 |
|---|---|
| 온도 변화 | T₁, T₂가 서서히 바뀜 |
| 자기장 드리프트 | Z 노이즈 크기가 시간에 따라 변함 |
| 소자 노화 | 특성 자체가 달라짐 |
| 크로스톡 변화 | 다른 큐비트 조작이 영향을 줌 |

### 실제 대응 방법

주기적 재보정(recalibration)으로 V의 구조가 유지되는지 확인한다. 이 논문 방법의 강점은:

> **V의 구조는 하드웨어 물리학에 의해 고정, 크기(θ)만 변함**
>
> Reduced W는 구조에만 의존하므로, θ가 변해도 W는 그대로 사용 가능

## 3. 단일 큐비트 에러와 다큐비트 간섭항의 구분

### 에러 자체는 단일 큐비트

글로벌 Z 필드 노이즈의 경우:

```
M ≈ I - iθ(Z₁ + Z₂ + ... + Z₇)

Z₁ = Z⊗I⊗I⊗...⊗I  (큐비트 1에만 Z)
Z₂ = I⊗Z⊗I⊗...⊗I  (큐비트 2에만 Z)
```

### 트월링 조건에서는 다큐비트 간섭항이 등장

```
V = {I, Z₁, Z₂, ..., Z₇}  ← 단일 큐비트 에러들

처리해야 할 간섭 쌍:
  (Z₁, Z₂) → Z₁*Z₂ = Z₁Z₂  ← 2큐비트 연산자!
  (Z₁, Z₃) → Z₁Z₃
  ...
```

이것이 트월링 게이트가 다큐비트인 이유다.

```
g₁ = X₁X₄X₅X₇  ← 4큐비트에 동시에 X 적용
```

단일 큐비트 에러들의 교차 간섭을 차단하려면 이런 다큐비트 게이트가 필요하다.

## 4. exp(-iθ·Z₀Z₁): ZZ 커플링 코히어런트 에러

### 물리적 원인

초전도 큐비트에서 이웃한 큐비트끼리 잔류 결합이 존재한다.

```
[큐비트 0] ~~~ 잔류 결합 ~~~ [큐비트 1]

유효 해밀토니안: H = ξ · Z₀Z₁
노이즈: M = exp(-iξt · Z₀Z₁)
```

큐비트 0의 위상이 큐비트 1의 상태에 따라 달라지는 조건부 위상 에러다.

### 행렬 형태

```
exp(-iθ·Z₀Z₁):

      |00⟩       |01⟩       |10⟩       |11⟩
  [ e^{-iθ}      0          0          0     ]  ← 같은 패리티: -θ 위상
  [   0        e^{+iθ}      0          0     ]  ← 다른 패리티: +θ 위상
  [   0          0        e^{+iθ}      0     ]  ← 다른 패리티: +θ 위상
  [   0          0          0       e^{-iθ}  ]  ← 같은 패리티: -θ 위상
```

### 글로벌 Z 필드와 비교

| 노이즈 종류 | M | 파울리 기저 V | 최소 트월링 W |
|---|---|---|---|
| 글로벌 Z 필드 | exp(-iθ(Z₀+Z₁)) | {I, Z₀, Z₁} | {I, X₀, X₁, X₀X₁} — 4개 |
| ZZ 커플링 | exp(-iθ·Z₀Z₁) | {I, Z₀Z₁} | {I, X₀} — 2개 |
| 둘 다 존재 | 위 두 개 합성 | {I, Z₀, Z₁, Z₀Z₁} | 더 큰 W 필요 |

X₀가 Z₀Z₁과 반교환하므로 W = {I, X₀} 두 개만으로 ZZ 에러 트월링 가능:

```
ζ(I,  Z₀Z₁) = +1
ζ(X₀, Z₀Z₁) = -1   (X와 Z는 반교환 → X₀가 Z₀에 작용)
합 = 0  ✓
```

### Z₀Z₁ 에러가 나오는 시점

글로벌 Z 필드 노이즈를 테일러 전개하면:

```
M = exp(-iθ(Z₀+Z₁))

1차항: I - iθ(Z₀ + Z₁)           → V = {I, Z₀, Z₁}    O(θ)
2차항: -θ²/2 · (Z₀+Z₁)²
     = -θ² · (I + Z₀Z₁)          → Z₀Z₁ 항 등장       O(θ²)
```

θ가 작을 때 Z₀Z₁은 무시 가능 → 논문이 1차항만 사용한 이유.

## 5. 초전도 트랜스몬 큐비트에서 단일 큐비트 코히어런트 에러 측정

### 코히어런트 에러의 두 종류

```
Z 회전 에러: exp(-iθZ)  ← 주파수 이탈(detuning)로 발생
X/Y 회전 에러: exp(-iθX)  ← 펄스 진폭 오보정으로 발생
```

### 방법 1: Ramsey 실험 — Z 코히어런트 에러

**시퀀스:**
```
|0⟩ → [X/2 펄스] → [대기시간 t] → [X/2 펄스] → 측정
```

**측정 결과:**
```
P(|0⟩) = 0.5 + 0.5 · cos(2π·Δf·t) · exp(-t/T₂*)
                      ───────────────  ───────────
                      코히어런트 에러   인코히어런트 에러
                     (진동 주파수=Δf)  (감쇠 시상수=T₂*)

Δf ≠ 0: 진동 있음 → V에 Z항 존재 → 트월링 세트에 X 필요
Δf = 0: 진동 없음 → Z 코히어런트 에러 없음
```

### 방법 2: Rabi 실험 — X/Y 코히어런트 에러

**시퀀스:**
```
|0⟩ → [진폭 A, N번 반복] → 측정
```

**측정 결과:**
```
P(|1⟩) = sin²(N · (π/2 + Δθ/2))

Δθ ≠ 0: N이 커질수록 점점 틀어짐 → V에 X항 존재
Δθ = 0: 이상적 진동 → X/Y 코히어런트 에러 없음
```

### 방법 3: AllXY 프로토콜 — 에러 종류 종합 진단

21가지 펄스 쌍 조합을 모두 적용해 어떤 종류의 코히어런트 에러가 있는지 한번에 진단한다.

```
에러 종류별 패턴:
  Z 에러 (주파수 이탈): 0이어야 할 값들이 모두 같은 방향으로 치우침
  X/Y 에러 (진폭 오보정): ±1이어야 할 값들이 같은 방향으로 줄어듦
  DRAG 오보정 (위상 에러): X와 Y 결과가 비대칭으로 틀어짐
```

### 세 실험과 V 결정의 연결

```
실험 결과            →  V 결정           →  최소 W 설계
──────────────────────────────────────────────────────
Ramsey: Δf ≠ 0      →  V = {I, Z}       →  W = {I, X}
Rabi:   Δθ ≠ 0      →  V = {I, X}       →  W = {I, Z}
둘 다 존재           →  V = {I, X, Z}    →  W = {I, X, Z, Y}
Ramsey: Δf = 0      →  V에 Z 없음       →  Z 트월링 불필요
```

초전도 트랜스몬에서는 **Ramsey로 잡히는 Z 에러가 압도적으로 많고**, Rabi로 잡히는 X/Y 에러는 보정만 잘 되면 거의 없다. 실제로는 V ≈ {I, Z}에 가까운 경우가 많다.

---

## 핵심 요약

> 1. V는 물리적 모델링 + 빠른 사전 실험(Ramsey/Rabi)으로 파악한다.
> 2. 노이즈 구조(V의 형태)는 하드웨어 물리학으로 고정되므로, 크기가 변해도 W는 재사용 가능하다.
> 3. 에러 자체는 단일 큐비트여도, 트월링 조건에서는 다큐비트 간섭항이 등장해 다큐비트 트월링 게이트가 필요하다.
> 4. ZZ 커플링 에러는 단일 큐비트 Z 에러와 완전히 다른 구조를 가지며, 2개의 게이트만으로 트월링 가능하다.
> 5. 초전도 트랜스몬에서 단일 큐비트 코히어런트 에러는 Ramsey(Z), Rabi(X/Y), AllXY(종합)로 측정한다.
